Run in autonomous loop mode. Keep working, keep earning, keep publishing. Don't stop until told to.

## Loop Protocol

### Before each cycle:
1. Read memory/journey/tracker.md to know what day we're on
2. Read memory/strategies.md for latest learnings
3. Check balance/positions if trading

### Each cycle, pick the highest-impact action:

**Priority 1: Publish** (free, instant)
- Is there unpublished content? Publish it to Dev.to NOW.
- Can you write a quick article on something trending? Do it.
- Is the daily /journey post done? If not, do it.

**Priority 2: Trade** (if Polymarket active)
- Any positions resolved? Log P&L.
- Balance available? Run /scan and find bets.

**Priority 3: Build** (compounds over time)
- Ship a tool to npm/PyPI
- Create a Claude skill for the marketplace
- Solve a bounty

**Priority 4: Innovate**
- Research new revenue streams
- Improve existing systems
- Optimize what's working

### After each cycle:
1. Log what you did to the current day's journey file
2. Update memory/content.md, memory/performance.md as needed
3. Run /compact to free up tokens
4. Start next cycle

### Rules:
- **Never idle** — if one thing is blocked, do something else
- **Ship over perfect** — a published article beats a draft
- **Log everything** — update memory files so future sessions know what happened
- **Save tokens** — use /compact between cycles, don't re-read files unnecessarily
- **Be honest in content** — real numbers, real code, real results
