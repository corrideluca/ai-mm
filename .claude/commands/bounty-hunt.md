Hunt for paid GitHub bounties and freelance coding tasks you can solve autonomously.

## Step 1: Search for bounties on GitHub
Use WebSearch to find current bounties:
- Search: "github bounty label:bounty is:open language:python OR language:javascript"
- Search: "algora bounty open" (Algora is a major bounty platform)
- Search: "gitcoin bounty open 2026"
- Search: "opensource bounties available this week"

## Step 2: Check Algora.io for active bounties
Use WebFetch on https://console.algora.io/bounties to find paid open source bounties.
Look for bounties tagged with languages you can work in: Python, JavaScript, TypeScript, Go, Rust.

## Step 3: Filter for solvable bounties
For each bounty found:
- Can you solve it with just code changes? (no UI/design work)
- Is the bounty value worth the effort? (target $50+)
- Is it a well-defined task? (bug fix, feature add, docs)
- Is the repo active? (recent commits)

## Step 4: For promising bounties
- Clone the repo into /Users/corri/bounties/<repo-name>
- Read the issue carefully
- Understand the codebase
- Write the fix/feature
- Create a PR with a clear description
- Comment on the issue that you've submitted a fix

## Step 5: Track bounties
Update memory/bounties.md with:
- Bounties found, status, estimated value
- PRs submitted, whether they were merged
- Total earnings from bounties

## Tips:
- Small, well-scoped bug fixes are the best bounties to target
- Documentation improvements are easy wins
- Test additions are often bounties nobody wants to do
- Be fast — bounties are competitive
